# TO-DOS

## Code Review Findings - Must Fix Before Merge - 2026-02-12 09:27

- **Fix backend hasRole() to exclude Moderators from Admin** - Backend role hierarchy does not match the frontend fix. **Problem:** Frontend `hasRole` was fixed to prevent Moderators from being treated as Admin (`&& role !== 'Admin'`), but backend `backend/lib/auth.ts` still grants Moderators all roles including Admin. Moderators can bypass frontend restrictions via direct API calls to perform Admin-only operations (update site config, manage fantasy config, toggle user accounts, list users). **Files:** `backend/lib/auth.ts:38`, `frontend/src/contexts/AuthContext.tsx:154`, `frontend/src/services/cognito.ts:286`. **Solution:** Change backend `hasRole()` to: `if (context.groups.includes('Moderator') && !requiredRoles.includes('Admin')) return true;`

- **Add timeout to recordResult Lambda** - Awaited recalculations risk Lambda timeout. **Problem:** Changed from fire-and-forget to `await Promise.allSettled(...)` for `triggerRankingRecalculation()` and `recalculateCosts()`, which perform full-table scans and dozens of DynamoDB writes (3-8+ seconds). The `recordResult` function has no explicit timeout (defaults to 6s). Core transaction succeeds but Lambda times out before responding. **Files:** `backend/functions/matches/recordResult.ts:672-683`, `backend/serverless.yml` (recordResult function definition). **Solution:** Immediate: add `timeout: 29` to recordResult in serverless.yml. Better: decouple via async Lambda invocation (`InvocationType: 'Event'`) or SQS.

- **Add ADMIN_SETUP_KEY to CI/CD workflows** - Default fallback is guessable and publicly visible in repo. **Problem:** `ADMIN_SETUP_KEY` defaults to `'default-dev-key-change-in-production'` and is not referenced in either deploy workflow. The `/auth/setup` endpoint is unauthenticated, so anyone who reads the repo can create Admin users. **Files:** `backend/serverless.yml:31`, `.github/workflows/deploy-prod.yml`, `.github/workflows/deploy-dev.yml`. **Solution:** Add `ADMIN_SETUP_KEY: ${{ secrets.ADMIN_SETUP_KEY }}` to both deploy workflows. Consider removing the default fallback entirely.

- **Fix CORS preflight/response origin mismatch** - `cors: true` returns `*` for OPTIONS while Lambda returns stage-specific origin. **Problem:** Every function uses `cors: true` (generates `Access-Control-Allow-Origin: *` for preflight), but Lambda response headers now return stage-specific origins. Combined with `Access-Control-Allow-Credentials: true`, this is an invalid CORS configuration that browsers will reject for credentialed requests. **Files:** `backend/serverless.yml` (all function cors settings), `backend/lib/response.ts:4-6`. **Solution:** Replace `cors: true` with explicit CORS config referencing `${self:custom.allowedOrigin.${self:provider.stage}}` with `allowCredentials: true`.

- **Change CORS fallback from wildcard to production domain** - `|| '*'` combined with credentials header is invalid per CORS spec. **Problem:** `response.ts` has `process.env.ALLOWED_ORIGIN || '*'` with `Access-Control-Allow-Credentials: true`. Wildcard + credentials is rejected by browsers. Silent degradation if env var is unset. **Files:** `backend/lib/response.ts:5`. **Solution:** Change fallback to `'https://leagueszn.jpdxsolo.com'` instead of `'*'`.

- **Exclude source map files from S3 deployment** - Hidden source maps still uploaded to public bucket. **Problem:** `sourcemap: 'hidden'` generates .map files without URL references, but `aws s3 sync dist` still uploads them where they're accessible via guessable filenames. **Files:** `.github/workflows/deploy-prod.yml` (s3 sync step), `.github/workflows/deploy-dev.yml`. **Solution:** Add `--exclude "*.map"` to the `aws s3 sync` command.
